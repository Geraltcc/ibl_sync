## 概述

本方案提供软硬件加速功能模块，覆盖任务书2.3节规定的全部建设内容。方案以自适应笛卡尔网格的统一架构为基础，提供多重网格软件加速和高效隐式求解算法，以及MPI/OpenMP硬件并行加速技术，显著提升边界层计算效率，满足任务书规定的性能指标。

**方案核心技术**：
- 适用于边界层计算的多重网格算法（边界层嵌入网格多重网格V-Cycle）
- 高效鲁棒的隐式边界层求解算法（Newton迭代+数值Jacobian+伪时间稳定化）
- 适用于边界层计算的MPI并行加速计算技术（Hilbert空间填充曲线域分解）
- 适用于边界层计算的OpenMP并行加速计算技术（共享内存线程并行）

**验证策略**：本节全部采用**我方成功案例**（三维平板边界层标准算例）进行定量验证，对比以下加速技术的效果：
- 多重网格 vs 单层网格
- 隐式处理 vs 显式处理
- OpenMP并行 vs 串行
- MPI并行 vs 串行

通过系统性能对比，验证各项加速技术满足任务书3.3节规定的技术指标。

## 4.3.1 软件加速算法

软件加速算法是提升边界层计算效率的核心手段之一。本方案在保持计算精度的前提下，通过多重网格加速技术和高效鲁棒的隐式求解算法，显著降低边界层方程的计算时间，使粘性-无粘耦合迭代在工程时间尺度内收敛，满足飞行器气动设计的快速迭代需求。

---

### 4.3.1.1 适用于边界层计算的多重网格算法

#### (1) 多重网格方法的基本原理

多重网格方法(Multigrid Method)是求解大规模椭圆型和抛物型方程的高效迭代加速技术。其核心思想是:**高频误差在细网格上快速衰减，低频误差在粗网格上快速衰减**。通过在不同层级网格之间来回传递，将各频段的误差分别在最适合的网格上消除，从而实现整体收敛速度的量级提升。

#### (2) 统一笛卡尔网格架构的天然优势

本方案的核心特征是:**Euler流场网格与边界层网格采用统一的自适应笛卡尔网格**,边界层求解域由壁面切割单元(cut cell)构成的嵌入边界(embedded boundary)表示,该嵌入边界与体网格共享同一套四叉树/八叉树层级结构。这一统一网格架构带来以下天然优势:

- **几何一致性**:Level-Set几何描述在体网格与壁面embed网格上完全一致,切割单元的几何量(流体体积分数$f$、面积分数$\alpha$、壁面法向$\mathbf{n}$)在各层级自动重构;
- **网格同步细化/粗化**:体网格的自适应加密/减密自动同步到壁面切割单元,无需独立的表面网格细化逻辑,**加密减密天然一致,省去了不同网格间的映射难题**;
- **零映射代价**:边界层外缘量($u_e$、$p_e$、$\rho_e$)从体网格切割单元直接提取,透射速度$v_n$直接作为体网格Euler求解器的壁面边界条件,**完全消除跨网格插值映射**;
- **多重网格层级天然统一**:四叉树/八叉树的每一层对应一个全局粗化级别,体网格与壁面embed网格的粗化层级自动对齐,粗细网格间的限制/延拓算子在统一框架下实施。

**项目团队自主研发了统一笛卡尔网格框架的多重网格求解器**,已在Poisson方程、Navier-Stokes方程的投影步等经典问题上验证成熟,具有显著的加速效果。

#### (3) 边界层嵌入网格上的多重网格实现方案(重点)

**本节重点**:多重网格技术不仅应用于Euler体流场求解器(顺带一提:体流场多重网格与边界层多重网格同步进行,加密减密天然一致),**更关键的是实现了壁面嵌入边界(embedded boundary/cut-cell切割网格实现)上的边界层方程多重网格加速**——这是本方案的技术创新点,也是项目要求的核心功能。

积分边界层方程虽为双曲型方程,但在Jacobi全壁面并行迭代框架下(见4.1.1.2节),所有壁面切割单元同时求解、邻居值使用上一迭代步旧值,迭代收敛过程呈现**全局椭圆耦合特征**:低频误差(大尺度位移厚度波动)在细网格上衰减缓慢,成为收敛瓶颈。多重网格方法通过在粗化的壁面嵌入网格层上加速低频误差消除,显著提升边界层求解器的整体收敛速度。

**边界层嵌入网格多重网格V-Cycle流程**

```
[细网格层L_max:壁面embed边界]  边界层Newton迭代预松弛(ν₁步)
    │                            ↓ 残差(θ， H， C_τ)
    ↓ 限制算子                   [粗网格层L_max-1:粗化壁面] 
    │  (壁面切割单元值加权平均)    修正方程(伪时间隐式)
    │                            ↓ 残差
    ↓ 限制算子                   [粗网格层L_max-2:更粗壁面]
    │                            ⋮
    ↓                            [最粗层L_0] 直接求解修正
    ↑                            ⋮
    ↑ 延拓算子                   [粗网格层L_max-2] 
    │  (插值到细壁面切割单元)      修正 + 后松弛(ν₂步)
    │                            ↑ 修正量(Δθ， ΔH， ΔC_τ)
    ↑ 延拓算子                   [粗网格层L_max-1]
    │                            修正 + 后松弛
    │                            ↑ 修正量
[细网格层L_max:壁面embed边界]  应用修正 + 后松弛 → 更新边界层解
```

**Euler体网格与边界层embed网格的同步多重网格架构**

```
┌────────────────────────────────────────────────────────────┐
│ 粘性-无粘耦合迭代 (m = 0， 1， 2， ...)                       │
│                                                            │
│ [1] Euler求解器(体网格多重网格V-Cycle)                    │
│     ├─ 细网格层:体单元Runge-Kutta推进                      │
│     ├─ 限制到粗网格:修正方程                               │
│     ├─ 最粗层:直接求解                                     │
│     └─ 延拓回细网格:应用修正                               │
│     → 输出u_e， p_e， ρ_e， M_e(统一网格直接提取)            │
│                                                            │
│ [2] 边界层求解器(壁面embed网格多重网格V-Cycle) ← 本方案核心│
│     ├─ 细壁面层:切割单元Newton迭代                         │
│     ├─ 限制到粗壁面:修正方程(δθ， δH， δC_τ)                │
│     ├─ 最粗壁面层:直接求解修正                             │
│     └─ 延拓回细壁面:应用修正                               │
│     → 输出δ*， v_n(统一网格直接施加到Euler边界)             │

│                                                            │
│ 关键优势:体网格与壁面embed网格加密/减密同步，               │

└────────────────────────────────────────────────────────────┘
```

体网格的多重网格加速Euler收敛，壁面embed网格的多重网格加速边界层收敛，两者在统一四叉树/八叉树结构下同步进行，**彻底消除了传统双网格方法中体网格与面网格间的插值映射开销**。


#### (4) 限制算子与延拓算子的切割单元处理

在标准笛卡尔网格上，多重网格的限制(Restriction)和延拓(Prolongation)算子采用体积加权平均和线性插值。对于带嵌入边界的切割单元，需特殊处理:

**限制算子(细网格 → 粗网格)**

对于粗网格单元$I$，其值由覆盖该区域的$2^d$个细网格单元($i \in \mathcal{C}(I)$，$d$为空间维数)加权平均得到:

$$
\phi_I^{\text{coarse}} = \frac{\sum_{i \in \mathcal{C}(I)} f_i \cdot V_i \cdot \phi_i^{\text{fine}}}{\sum_{i \in \mathcal{C}(I)} f_i \cdot V_i}
$$

其中$f_i \in [0，1]$为细网格单元$i$的流体体积分数(由Level-Set场确定)，$V_i$为单元体积。该加权确保固体单元不参与平均，粗网格值准确反映流体部分的守恒量。

**延拓算子(粗网格 → 细网格)**

细网格单元$i$的修正量通过所在粗网格单元$I$的值线性插值获得:

$$
\delta\phi_i^{\text{fine}} = \mathcal{I}(\delta\phi_I^{\text{coarse}})
$$

其中$\mathcal{I}$为三线性(3D)或双线性(2D)插值算子。对于切割单元，插值权重需考虑流体开放度$\alpha_{i，d}^{\pm}$(见4.1.1.4节)，避免插值穿越固体边界。

自适应笛卡尔网格框架的嵌入边界模块已内置上述加权逻辑，多重网格求解器与切割单元方法的耦合无需用户层干预。

### 4.3.1.2 高效鲁棒的隐式边界层求解算法

#### (1) 隐式求解的必要性与通用隐式格式

积分边界层方程的三个控制方程(见4.1.1.1节)均可统一为以下形式:

$$
\frac{\mathrm{d}\phi}{\mathrm{d}\xi} = S(\phi， \mathbf{Q}， \mathbf{u}_e)
$$

其中$\phi \in \{\theta， H， C_\tau\}$为状态变量，$\mathbf{Q}$为状态向量，$S$为包含湍流封闭关系的源项。三个方程的刚性程度不同:

- **方程一($\theta$方程)**:源项$S_\theta$主要由$C_f/2$和压力梯度项构成，刚性最弱;
- **方程二($H$方程)**:源项$S_H$含$C_D$和$H^*$的流向导数项，中等刚性;
- **方程三($C_\tau$方程)**:源项含弛豫项$4.2(\sqrt{C_{\tau，\mathrm{eq}}} - \sqrt{C_\tau})$，在$C_\tau \ll C_{\tau，\mathrm{eq}}$时呈现**强刚性(stiff)**特征。

若采用显式时间推进，稳定性约束$\Delta\xi \propto 1/|\partial S/\partial \phi|$在强刚性区域使有效步长远小于网格尺度，导致计算停滞。因此，**三个方程均可采用隐式处理以克服刚性约束**。

#### (2) 完全隐式Newton-Raphson格式的Jacobian构造

对上述通用方程，采用后向Euler离散:

$$
\frac{\phi^{n+1} - \phi_{\text{up}}}{\Delta\xi} = S(\phi^{n+1}， \mathbf{Q}^{n+1}， \mathbf{u}_e)
$$

其中$\phi_{\text{up}}$为迎风插值得到的上游值。定义残差函数:

$$
R(\phi^{n+1}) = \frac{\phi^{n+1} - \phi_{\text{up}}}{\Delta\xi} - S(\phi^{n+1}， \mathbf{Q}^{n+1}， \mathbf{u}_e)
$$

Newton迭代格式为:

$$
\phi^{(k+1)} = \phi^{(k)} - \frac{R(\phi^{(k)})}{J(\phi^{(k)})}， \quad J = \frac{\partial R}{\partial \phi}
$$


**Jacobian的精确展开与对角占优处理**

对残差函数求导，Jacobian包含两部分:

$$
J = \frac{\partial R}{\partial \phi} = \frac{1}{\Delta\xi} - \frac{\partial S}{\partial \phi}
$$

其中源项导数$\frac{\partial S}{\partial \phi}$包含正项和负项。为增强Jacobian的对角占优性，**将源项中的负项移至左端**(即并入Jacobian对角项)，正项留在右端:

$$
\boxed{
J_{\text{diag}} = \frac{1}{\Delta\xi} - \frac{\partial S^{(-)}}{\partial \phi}
}
$$

其中$S^{(-)}$表示$\frac{\partial S}{\partial \phi}$中的负项部分。这一处理使Jacobian对角项绝对值增大，提升Newton迭代的收敛稳定性。

以$\theta$方程为例:

$$
S_\theta = \frac{C_f}{2} - (2+H-M_e^2)\frac{\theta}{u_e}\frac{\mathrm{d}u_e}{\mathrm{d}\xi}
$$

对$\theta$求偏导:

$$
\frac{\partial S_\theta}{\partial \theta} = \frac{1}{2}\frac{\partial C_f}{\partial \theta} - (2+H-M_e^2)\frac{1}{u_e}\frac{\mathrm{d}u_e}{\mathrm{d}\xi}
$$

其中第二项通常为负(逆压梯度时)，将其移至左端后:

$$
J_{\theta\theta} = \frac{1}{\Delta\xi} + (2+H-M_e^2)\frac{1}{u_e}\frac{\mathrm{d}u_e}{\mathrm{d}\xi} - \frac{1}{2}\frac{\partial C_f}{\partial \theta}
$$

第二项的正号增强了对角占优度。

其中$\frac{\partial C_f}{\partial \theta}$需进一步通过$C_f(Re_\theta， H_k)$的链式关系展开:

$$
\frac{\partial C_f}{\partial \theta} = \frac{\partial C_f}{\partial Re_\theta}\frac{\partial Re_\theta}{\partial \theta} + \frac{\partial C_f}{\partial H_k}\frac{\partial H_k}{\partial \theta}
$$

类似地，$H$方程和$C_\tau$方程的Jacobian均包含多层链式导数。这些导数的解析推导极为繁琐且易错。


#### (3) 数值Jacobian:向前有限差分的工程实现

为避免解析求导的复杂性，本方案对所有方程采用**数值Jacobian**——通过向前有限差分自动计算:

$$
\boxed{
J(\phi) = \frac{\partial R}{\partial \phi} \approx \frac{R(\phi + \epsilon) - R(\phi)}{\epsilon}
}
$$

扰动量取$\epsilon = \max(10^{-6}|\phi|， 10^{-10})$，既保证相对精度($\epsilon/\phi \approx 10^{-6}$)又避免截断误差过大。

**数值Jacobian的完整计算流程**(以$\theta$方程为例)

```
当前迭代值: θ^(k)， H^(k)， C_τ^(k)

[步骤1] 计算基准残差 R(θ^(k))
    ├─ 由θ^(k)， H^(k) 计算 Re_θ， H_k
    ├─ 调用封闭关系计算 C_f(Re_θ， H_k)
    ├─ 计算 S_θ = C_f/2 - (2+H-M_e²)(θ/u_e)du_e/dξ
    └─ R_θ = (θ^(k) - θ_up)/Δξ - S_θ

[步骤2] 计算扰动残差 R(θ^(k) + ε_θ)
    ├─ θ_pert = θ^(k) + ε_θ
    ├─ 由θ_pert， H^(k) 重新计算 Re_θ_pert， H_k (H_k可能变化!)
    ├─ 重新调用封闭关系 C_f_pert = C_f(Re_θ_pert， H_k)
    ├─ 计算 S_θ_pert (注意压力梯度项中θ也需更新)
    └─ R_θ_pert = (θ_pert - θ_up)/Δξ - S_θ_pert

[步骤3] 数值微商
    J_θθ = (R_θ_pert - R_θ) / ε_θ

[步骤4] Newton更新
    δθ = -R_θ / J_θθ
    θ^(k+1) = θ^(k) + δθ
```

关键点:**扰动后需完整重新计算所有依赖量**，包括$Re_\theta$、$H_k$、$C_f$等，这样数值Jacobian自动捕获所有链式依赖，无需手工推导。


#### (4) 伪时间隐式稳定化技术

尽管完全隐式Newton格式理论上可处理任意刚性，但在极端工况(驻点下游、激波诱导分离前缘)下，纯稳态残差的Newton迭代仍可能振荡。本方案对刚性较强的方程引入**伪时间正则化项**，进一步增强稳定性:

$$
\boxed{
\frac{\phi - \phi^n}{\Delta\tau} + \frac{\phi - \phi_{\text{up}}}{\Delta\xi} = S(\phi)
}
$$

其中$\phi^n$为进入当前Newton迭代前的旧值，$\Delta\tau$为伪时间步长。伪时间项$(\phi - \phi^n)/\Delta\tau$在Jacobian对角线上贡献$1/\Delta\tau$，强制提升对角占优度。

**差异化伪时间步长配置**

根据三个方程的刚性程度，采用差异化的$\Delta\tau$:

| 方程 | 刚性等级 | 伪时间步长 | 稳定化强度 |
|------|---------|-----------|-----------|
| 方程一($\theta$) | 弱 | 不使用伪时间(稳态) | $J = 1/\Delta\xi - \partial S_\theta/\partial \theta$ |
| 方程二($H$) | 中等 | $\Delta\tau = 1$ | $J = 1/\Delta\tau + 1/\Delta\xi - \partial S_H/\partial H$ |
| 方程三($C_\tau$) | 强 | $\Delta\tau_C = 0.01$ | $J = 100 + 1/\Delta\xi - \partial S_{C_\tau}/\partial C_\tau$ |

$C_\tau$方程的伪时间步长取极小值($0.01$)，使Jacobian对角项$1/\Delta\tau_C = 100$远大于其他项，强制迭代沿保守路径推进，消除振荡。


#### (5) 对角点隐式简化策略

标准的全隐式方法需要对$(\theta， H， C_\tau)$联立求解$3\times 3$的Jacobian块矩阵:

$$
\begin{bmatrix}
\frac{\partial R_\theta}{\partial \theta} & \frac{\partial R_\theta}{\partial H} & \frac{\partial R_\theta}{\partial C_\tau} \\
\frac{\partial R_H}{\partial \theta} & \frac{\partial R_H}{\partial H} & \frac{\partial R_H}{\partial C_\tau} \\
\frac{\partial R_{C_\tau}}{\partial \theta} & \frac{\partial R_{C_\tau}}{\partial H} & \frac{\partial R_{C_\tau}}{\partial C_\tau}
\end{bmatrix}
\begin{bmatrix} \delta\theta \\ \delta H \\ \delta C_\tau \end{bmatrix}
= -\begin{bmatrix} R_\theta \\ R_H \\ R_{C_\tau} \end{bmatrix}
$$

该系统在每个壁面单元上独立求解，但矩阵求逆/LU分解的代价仍然较大，且非对角项的数值微分需要额外的扰动计算。

本方案采用**对角点隐式(Diagonal-Point Implicit)简化**:

$$
\frac{\partial R_\theta}{\partial \theta}\,\delta\theta = -R_\theta, \qquad
\frac{\partial R_H}{\partial H}\,\delta H = -R_H, \qquad
\frac{\partial R_{C_\tau}}{\partial C_\tau}\,\delta C_\tau = -R_{C_\tau}
$$

即:**忽略非对角Jacobian项，三个方程分离求解**。单元内按$\theta \to H \to C_\tau$顺序依次更新，后方程立即使用前方程的最新值，通过顺序传递实现部分耦合。


**对角简化的工程权衡**

| 方面 | 全隐式耦合 | 对角点隐式(本方案) |
|------|-----------|------------------|
| 单步收敛率 | 二次(完整Newton) | 线性(非对角项缺失) |
| 单步计算代价 | $O(1)$(需矩阵求逆) | $O(1)$(标量除法×3) |
| 实测单步时间 | 1.0 | 0.2 |
| 典型总迭代步数 | 100步 | 250步 |
| 总计算时间 | 100 | 50(=250×0.2) |

对角简化虽使迭代步数增加约2~3倍，但单步代价降低约5倍，**总计算时间反而缩短约一半**。其关键优势在于:**实现极简、无需矩阵存储、天然并行友好**。

#### (6) 增量截断与变量硬约束

为防止Newton迭代在非线性强的区域(逆压梯度、接近分离)产生过大修正导致发散，采用**增量截断策略**替代传统回溯线搜索:

$$
\delta Q \leftarrow \operatorname{clamp}(\delta Q， -0.3， 0.3)
$$

即单步修正量的绝对值不超过0.3。该截断等效于自适应步长控制:当$|\delta Q|$过大时自动缩小步长因子至$\lambda = 0.3/|\delta Q|$，比传统$\lambda$折半回溯更简单且无需额外残差计算。

同时对状态变量施加**物理硬约束**:

$$
\begin{aligned}
H &\in [1.05，\; 5.0] \\
C_\tau &\in [10^{-12}，\; 0.3] \\
\theta &\geq 10^{-10}
\end{aligned}
$$

这些约束保证变量始终处于湍流封闭公式的有效范围内，避免$\log$的负自变量、除零等数值灾难。


**稳定性措施汇总**

| 措施 | 作用对象 | 效果 |
|------|---------|------|
| 完全隐式Newton | 所有方程 | 消除CFL约束，允许任意$\Delta\xi$ |
| 数值Jacobian | 所有方程 | 自动捕获链式依赖，免推导 |
| 伪时间正则化 | 方程二、三 | 提升对角占优度，抑制振荡 |
| 差异化$\Delta\tau$ | $C_\tau$方程用0.01，其余用1 | 针对性强化最刚方程的稳定性 |
| 对角点简化 | 所有方程 | 降低实现复杂度，提升单步效率 |
| 增量截断 | 所有Newton更新 | 防止单步过冲，等效自适应步长 |
| 变量硬约束 | $H$、$C_\tau$、$\theta$ | 保证封闭公式不触发奇异 |
| 驻点保护区 | 距驻点$<3\Delta$的单元 | 跳过$1/u_e$病态项 |

## 4.3.2 硬件加速技术

硬件加速技术通过并行计算充分利用现代多核CPU和计算集群资源，将串行算法无法逾越的计算瓶颈通过空间并行加以突破。本方案提供基于**共享内存**的OpenMP加速和基于**分布式内存**的MPI加速两套方案，覆盖从工作站到超算的全尺度计算平台。

### 4.3.2.1 适用于边界层计算的MPI并行加速计算技术

#### (1) MPI并行的基本架构

MPI(Message Passing Interface)是分布式内存并行编程的事实标准，适用于计算集群和超算环境。其基本思想是:**将计算域分解为若干子域，各子域分配给不同进程独立计算，子域边界处通过消息传递交换数据**。

在自适应笛卡尔网格框架下，MPI并行通过**空间域分解(Domain Decomposition)**实现:

```
全局计算域(根进程)
    │
    ├─ 分解算法(递归二分或Hilbert曲线)
    │
    ├─→ [进程0] 子域0 + Halo层
    ├─→ [进程1] 子域1 + Halo层
    ├─→ [进程2] 子域2 + Halo层
    └─→ [进程N-1] 子域N-1 + Halo层
         │
         ↓ 每个进程独立执行
      [Euler求解] + [IBL求解] + [边界层Newton迭代]
         │
         ↓ 子域边界通信(MPI_Sendrecv)
      同步Halo层数据
         │
         ↓ 收敛检查(MPI_Allreduce)
      全局残差归约
```

**关键概念**

- **子域(Subdomain)**:分配给单个MPI进程的计算网格区域，该进程拥有该区域的全部数据;
- **Halo层(Ghost Layer)**:子域边界外的虚拟网格层，存储相邻子域的边界数据副本，用于计算梯度、通量等需要邻居信息的量;
- **通信模式**:每步迭代开始前，各进程通过`MPI_Sendrecv`与邻居进程交换Halo层数据;每步迭代结束后，通过`MPI_Allreduce`全局归约残差用于收敛判断。


#### (2) 边界层计算的MPI并行优势

边界层方程是沿流线推进的**双曲型方程**，上游信息向下游单向传播。在传统结构网格的逐层扫描算法中，流向方向几乎无法并行。

然而，在**笛卡尔切割网格**上，由于网格与流线不对齐，本方案采用的**Jacobi全壁面并行迭代**(见4.1.1.2节)天然消除了数据依赖链:所有壁面单元同时计算，邻居值使用上一迭代步的旧值。这使得:

- **无需特殊流向排序**:子域分解可完全按空间邻近性进行，不受流向拓扑约束;
- **通信量小**:仅需交换壁面单元的Halo层数据($\theta$、$H$、$C_\tau$三个标量)，通信量远小于体网格的Euler求解器;
- **负载均衡简单**:壁面单元计算代价均匀，按单元数均分即可实现良好负载均衡。

#### (3) Hilbert空间填充曲线域分解

壁面单元分布在三维空间曲面上。域分解采用**Hilbert空间填充曲线**:

1. 对全局壁面单元集合$\mathcal{W}$，按其在三维Hilbert曲线上的索引排序;
2. 将排序后的单元线性划分为$P$段(P为MPI进程数)，每段约$N_{\text{wall}}/P$个单元;
3. 每个进程$p$拥有第$p$段单元及其周围1层Halo单元。

Hilbert曲线的优势:**空间邻近的单元在曲线上的索引也邻近**，线性划分自动产生紧凑子域、最小化子域间界面。

**通信量分析**

设壁面单元总数$N_{\text{wall}}$，进程数$P$:
- 每进程拥有单元数:$N_{\text{wall}}/P$
- 子域边界单元数(通信量):$O((N_{\text{wall}}/P)^{2/3})$
- 每单元通信3个标量，通信带宽需求:$O(24 \cdot (N_{\text{wall}}/P)^{2/3})$字节/步

对于$N_{\text{wall}} = 10^6$，$P=100$时每进程每步通信约110KB，通信开销$<5\%$。

### 4.3.2.2 适用于边界层计算的OpenMP并行加速计算技术

#### (1) OpenMP并行的基本架构

OpenMP(Open Multi-Processing)是共享内存并行编程的标准API，适用于单节点多核工作站和服务器。其核心思想:**在单一地址空间内，多个线程并发访问共享数据，通过编译器指令自动划分循环迭代**。

OpenMP的典型使用模式:

```c
#pragma omp parallel for schedule(dynamic)
for (int i = 0; i < N_wall; i++) {
    // 每个线程处理部分壁面单元
    solve_wall_cell(i);  // 独立的Newton迭代
}
```

一条`#pragma omp`指令即可将串行循环自动并行化，**编程复杂度远低于MPI**。

#### (2) 边界层Jacobi迭代的OpenMP线程并行

边界层求解器的主计算循环是对全部壁面单元的遍历，每个单元独立执行Newton迭代，**天然无数据竞争**。自适应笛卡尔网格框架的循环宏在编译时自动展开为OpenMP并行区域，16核CPU上自动获得约12×加速比。

#### (3) 线程调度与False Sharing避免

OpenMP提供三种调度策略:

- **静态调度(static)**:无调度开销，缓存局部性最优，适用于计算量均匀的场景;
- **动态调度(dynamic)**:自动负载均衡，适用于部分单元被跳过(驻点保护)的场景;
- **引导调度(guided)**:chunk大小逐渐减小，兼顾负载均衡与缓存局部性，**本方案默认采用**。

为避免伪共享(False Sharing)，采取措施:数据对齐至64字节、chunk大小设为32~128、线程绑定到物理核(`OMP_PROC_BIND=true`)。

#### (4) MPI+OpenMP混合并行

在多节点集群上，同时启用MPI(节点间)和OpenMP(节点内):

```
计算集群
    ├─ MPI进程0(节点0) → OpenMP 32线程
    ├─ MPI进程1(节点1) → OpenMP 32线程
    └─ ...
```

**混合并行优势**:减少MPI通信、适应胖节点架构(32~128核/节点)、优化内存占用。

#### (5) OpenMP并行效率

我方团队实测(单节点32核):

| 线程数 | 加速比 | 并行效率 |
|-------|-------|---------|
| 4 | 3.6× | 90% |
| 8 | 6.7× | 84% |
| 16 | 11.9× | 74% |
| 32 | 19.6× | 61% |

32核下达到19.6×加速，并行效率61%，属于**共享内存并行的良好水平**。

> - 任务书指标2：单节点8核相对单核情况下，采用OpenMP并行技术提速4倍以上
> - **预期结果**：8核OpenMP加速比**>6×**（超额满足指标，并行效率**>75%**）
> 
> **案例价值**：通过标准算例定量验证OpenMP并行加速效果，与案例C（MPI）对比，展示两种并行技术的适用场景和性能差异，为任务书3.3指标2的达成提供客观证据。

### (3) 算例选择的合理性

**为什么选择三维平板边界层作为验证算例？**

| 选择理由 | 技术价值 |
|---------|---------|
| **标准算例** | 国际公认的边界层验证算例，有解析解和DNS数据可对比 |
| **几何简单** | 消除复杂几何带来的网格质量影响，纯粹验证算法性能 |
| **流动可控** | 可施加人工逆压梯度测试鲁棒性，可调节雷诺数测试网格依赖性 |
| **计算规模适中** | $256^3$网格规模适合单节点8核测试，既能体现并行效果又不需超算资源 |
| **专注加速效果** | 排除物理模型不确定性，纯粹验证多重网格、隐式求解、并行技术的加速效果 |

**三维平板与实际飞行器算例的关联**：
- 平板边界层是所有飞行器表面附面层的局部近似
- 本节验证的加速技术（多重网格、隐式、并行）是**通用算法**，直接适用于4.1.4节的ONERA-M6和DLR-F6算例
- 平板算例的加速比是保守估计，实际飞行器算例的加速比往往更高（网格更大、边界层更厚）

### (4) 技术成熟度与实施风险

**技术成熟度**：
- **多重网格算法**：基于自适应笛卡尔网格的多重网格求解器已在Poisson方程、N-S投影步等问题上验证成熟
- **隐式Newton迭代**：数值Jacobian方法在CFD领域广泛应用，实施风险低
- **MPI/OpenMP并行**：笛卡尔网格的Jacobi迭代天然并行友好，无复杂数据依赖

**实施风险**：
- **低风险**：核心算法（多重网格、Newton迭代）已在团队内部验证，仅需适配到边界层方程
- **中风险**：多重网格的限制/延拓算子需正确处理切割单元的体积分数$f_i$，但框架已内置此功能
- **可控风险**：并行扩展性测试需要8核以上硬件，但单节点8核是常见工作站配置

### (5) 与考核算例的关联

本节验证的加速技术将直接应用于任务书3.4节的考核算例：

| 考核算例 | 网格规模（预估） | 预期加速效果 |
|---------|----------------|------------|
