| 要求项 | 任务书要求 | 本方案承诺 | 偏离情况 |

| 接口兼容性 | 兼容NNW-FAST | **提供接口适配层，完全兼容** | **无偏离** |

**具体实施方案**：

**（1）NNW-FAST接口分析**
基于甲方提供的接口文档，NNW-FAST的主要接口包括：
- **网格数据接口**：笛卡尔网格拓扑、Level-Set场、切割单元几何量
- **流场数据接口**：Euler求解器输出的守恒变量（密度、动量、能量）
- **边界条件接口**：壁面边界条件设置（速度、温度、吹吸）
- **控制参数接口**：求解器配置、收敛判据、输出控制

**（2）接口适配层设计**
```cpp
// 接口适配层类（Adapter Pattern）
class NNW_FAST_Adapter {
public:
    // 初始化：从NNW-FAST获取网格数据
    void initialize(const NNW_FAST_Mesh* mesh,
                   const NNW_FAST_LevelSet* levelset);
    
    // 每步迭代：获取Euler外缘量
    void get_edge_quantities(double* ue, double* pe, double* rhoe);
    
    // 每步迭代：设置边界层吹吸边界条件
    void set_transpiration_bc(const double* vn);
    
    // 输出：边界层结果写回NNW-FAST
    void export_boundary_layer_results(NNW_FAST_Output* output);
};
```

**（3）数据结构映射**
| NNW-FAST数据结构 | 本模块内部数据结构 | 映射方式 |
|-----------------|------------------|---------|
| `CartesianMesh` | `UnifiedMesh` | 指针共享（零拷贝） |
| `LevelSetField` | `φ(x)隐式场` | 直接引用 |
| `WallCellList` | `std::vector<WallCell>` | 一一对应 |
| `ConservativeVars` | `{ρ, ρu, ρE}` | 公式转换（守恒→原始变量） |

### 4.5.2.2 内存数据交换

**任务书要求**（第4.2节）：
> 功能模块之间通过内存进行数据交换，不能以文件形式通过硬盘进行数据交换。

**本方案响应内容**：

| 要求项 | 任务书要求 | 本方案承诺 | 偏离情况 |
|-------|-----------|----------|---------|
| 数据交换方式 | 仅内存，禁止文件 | **全部采用内存共享/MPI消息，零文件I/O** | **无偏离** |

**具体实施方案**：

**（1）单机内存共享**（Euler求解器↔IBL求解器）
- **统一网格架构的天然优势**：
  - Euler与IBL共享同一个`UnifiedMesh`对象（指针传递，零拷贝）
  - 外缘量提取：直接读取`mesh->cells[i].conservative_vars`
  - 边界条件设置：直接写入`mesh->wall_cells[k].transpiration_velocity`
- **实现方式**：
  ```cpp
  // Euler求解器侧
  void EulerSolver::solve_one_step() {
      // ... Euler方程求解 ...
      // 外缘量已自动存储在mesh->cells中，无需额外传递
  }
  
  // IBL求解器侧
  void IBLSolver::solve() {
      for (auto& wc : mesh->wall_cells) {
          // 直接读取外缘量（内存共享）
          double ue = wc.get_edge_velocity();
          // ... IBL求解 ...
          // 直接写入吹吸速度（内存共享）
          wc.transpiration_velocity = vn;
      }
  }
  ```

**（2）分布式内存通信**（MPI并行）
- **MPI消息传递**（跨节点数据交换）：
  - Halo层通信：`MPI_Sendrecv`交换边界单元状态量
  - 全局归约：`MPI_Allreduce`汇总残差、力系数等
  - 数据类型：MPI_DOUBLE（直接传递数值，无需序列化）
- **通信示例**：
  ```cpp
  // Halo层通信（边界单元数据交换）
  MPI_Sendrecv(send_buffer, n_send, MPI_DOUBLE, neighbor_rank, tag,
               recv_buffer, n_recv, MPI_DOUBLE, neighbor_rank, tag,
               MPI_COMM_WORLD, &status);
  
  // 全局残差归约
  double local_residual = compute_local_residual();
  double global_residual;
  MPI_Allreduce(&local_residual, &global_residual, 1, MPI_DOUBLE,
                MPI_SUM, MPI_COMM_WORLD);
  ```

**（3）严格禁止文件I/O（除输出）**
- **中间数据**：全部存储在内存（`std::vector`、指针、MPI缓冲区）
- **配置参数**：启动时从命令行或内存结构读取，不读取配置文件
- **唯一文件I/O**：最终结果输出（Tecplot、VTK格式），这是必需的后处理输出

**（4）内存管理策略**
- **智能指针**：`std::unique_ptr`、`std::shared_ptr`管理动态内存
- **内存池**：壁面单元数组预分配，避免频繁allocation
- **内存监控**：提供内存使用统计接口（可选功能）
